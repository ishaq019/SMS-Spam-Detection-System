


A robust machine learning pipeline and web application designed to classify SMS messages as **Spam** or **Ham (Legitimate)** with high precision. This project addresses the challenge of imbalanced text data by utilizing NLP techniques and a probabilistic classifier to minimize false positives.

## ğŸ“Œ Project Overview

Spam detection is a classic class-imbalance problem where legitimate messages significantly outnumber spam messages. In real-world scenarios, **Precision** is the critical metricâ€”flagging a legitimate message as spam (False Positive) is a severe failure, while missing a spam message (False Negative) is merely an inconvenience.

This project engineers a solution that prioritizes **Zero False Positives**, achieving a **100% Precision score** on the test dataset using Multinomial Naive Bayes.

## âœ¨ Key Features

* **End-to-End NLP Pipeline:** Automated cleaning, tokenization, stop-word removal, and stemming using NLTK.
* **Vectorization:** Implements **TF-IDF (Term Frequency-Inverse Document Frequency)** with a 3,000-feature limit to balance model performance and latency.
* **Probabilistic Modeling:** Utilizes **Multinomial Naive Bayes**, optimized for discrete frequency counts in text data.
* **Interactive Web Interface:** A user-friendly Streamlit frontend for real-time inference and testing.
